## 充值与商业化系统（待测试）

完整的商业化会员与充值系统，对标 Sub2API + Sub2APIPay 架构，支持直接部署收费运营。

+ [新增] 充值套餐系统：一次性额度套餐（体验/基础/标准/专业），购买后自动增加额度。
+ [新增] 订阅会员系统：月度/季度/年度订阅套餐，购买后创建订阅记录并发放当期额度。
+ [新增] 订单生命周期：完整状态机（PENDING→PAID→RECHARGING→COMPLETED），支持取消/超时/失败/退款。
+ [新增] 支付集成：EasyPay（易支付聚合）+ Stripe 双渠道，统一 Provider 接口，支持多实例。
+ [新增] 支付回调：验签 + CAS 原子状态更新 + 金额校验 + 幂等充值（recharge_code）。
+ [新增] 退款流程：先扣后退机制，网关退款失败自动回滚用户额度。
+ [新增] 兑换码系统：管理员批量生成、用户原子兑换（CAS 防并发）。
+ [新增] 优惠码系统：注册赠送额度。
+ [新增] 风控限额：三层限额（单笔/每日用户/每日渠道）+ 待支付上限 + 取消频率限制 + 订单超时。
+ [新增] 支付商配置加密：AES-256-GCM 加密存储，管理后台在线配置。
+ [新增] 收入仪表盘：日/总收入、订单数、成功率、趋势图、渠道分布、充值排行。
+ [新增] 管理后台：订单管理（筛选/退款）、套餐 CRUD、兑换码管理、支付配置。
+ [新增] 定时任务：订单超时清理 + 订阅过期标记。
+ [安全] 服务端读取套餐价格，不信任客户端金额。
+ [安全] 支付回调验签（EasyPay MD5 + Stripe webhook）+ PID 校验。
+ [安全] 退款 CAS 乐观锁防止竞态。

## v0.5.0 - 2026-06-23

多用户系统 + 服务端 AI 代理配额执行 + 管理员用户管理（参照 sub2api 权限模式）。

+ [新增] 用户认证系统：邮箱+密码注册/登录，bcrypt 哈希 + JWT（7天）+ HttpOnly Cookie。
+ [新增] 多用户权限体系：首个注册用户自动成为管理员；普通用户数据按 user_id 隔离。
+ [新增] 画布按用户完全隔离，用户间互不可见；素材库全局共享仅管理员可改。
+ [新增] 系统配置管理员模式：AI 渠道/Key 仅管理员可改，普通用户只读。
+ [新增] 管理员用户管理页面：增/删/改用户 + 配额分配（Ant Design 表格）。
+ [新增] 服务端 AI 代理 `/api/ai/[...path]`：所有 AI 流量经服务端转发，客户端不再持有 API Key。
+ [新增] 配额执行模型：quota（-1 管理员无限 / 0 新用户默认）/ used_quota，原子扣减 + 失败退还。
+ [新增] 成本计算：GET=0，图片生成=n（张数），其余 POST=1。
+ [新增] 剩余额度徽章（生成后实时刷新）+ 管理员头像角标。
+ [新增] SQLite 数据持久化（bind mount）+ Next.js proxy 中间件保护所有页面。
+ [安全] 服务端代理注入真实 Key，修复此前 API Key 泄露给所有客户端的问题。
+ [安全] `/api/config` 对非管理员脱敏 API Key（顶层 legacy 字段 + 各渠道均脱敏）。
+ [安全] 生产环境缺少 JWT_SECRET 时拒绝使用默认密钥。
+ [安全] 管理员降级时收回无限额度（quota -1 → 0），防止保留免费 AI 权限。
+ [修复] 注册接口增加密码≥6位 + 邮箱格式校验。
+ [修复] 配额并发放大窗口：上游 4xx 计费不退还，仅 5xx/网络错误退还。
+ [优化] 降低默认图片生成数，规避上游风控。
+ [防护] 禁止删除自己、禁止自我降级、保留至少一个管理员、级联清理画布。
